# PopKit v1.0.0-beta.1 Beta Tester Recruitment

This document outlines the strategy for recruiting and onboarding beta testers.

---

## Target Beta Tester Profile

### Ideal Candidates

**Primary Profile** (10-15 testers):
- Active Claude Code users (daily usage)
- Developers working on medium to large projects
- Comfortable with command-line tools
- Willing to provide detailed feedback
- Available for 4-8 week beta period

**Secondary Profile** (5-10 testers):
- New to Claude Code (fresh perspective)
- Different tech stacks (Python, TypeScript, Go, Rust, etc.)
- Different project types (monorepos, microservices, libraries)
- Various experience levels (junior to senior)

### Diversity Goals

- **Tech Stacks**: 40% TypeScript, 30% Python, 20% Other, 10% Polyglot
- **Project Sizes**: 30% Small (<100 files), 40% Medium (100-1000), 30% Large (1000+)
- **Experience**: 20% Junior, 50% Mid, 30% Senior
- **Geography**: Global distribution for timezone coverage
- **Company Size**: 40% Startups, 30% Mid-size, 30% Enterprise

## Application Process

### Application Form Questions

1. **Contact Information**
   - Name
   - Email
   - GitHub username (optional)
   - Twitter/X (optional)

2. **Development Background**
   - Primary tech stack(s)
   - Years of experience
   - Project types (web apps, APIs, libraries, etc.)
   - Current project size

3. **Claude Code Usage**
   - How long have you used Claude Code?
   - How often do you use it? (Daily / Weekly / Monthly)
   - What do you use it for?
   - Have you tried PopKit before?

4. **Beta Testing**
   - Why do you want to be a beta tester?
   - How much time can you commit per week? (2-4 hours ideal)
   - What's your preferred feedback method? (GitHub / Email / Slack)
   - Have you participated in beta programs before?

5. **Specific Interests**
   - Which plugins interest you most? (core / dev / ops / research / all)
   - What features are most important to you?
   - What would make this beta successful for you?

### Selection Criteria

**Priority Points System** (max 100):
- **Activity Level** (30 pts):
  - Daily Claude Code usage: 30pts
  - Weekly usage: 20pts
  - Monthly usage: 10pts

- **Project Relevance** (25 pts):
  - Large project (1000+ files): 25pts
  - Medium project: 15pts
  - Small project: 10pts

- **Feedback Quality** (20 pts):
  - Previous beta experience: 20pts
  - Active in open source: 15pts
  - No experience: 5pts

- **Tech Stack Diversity** (15 pts):
  - Underrepresented stack: 15pts
  - Common stack: 10pts

- **Time Commitment** (10 pts):
  - 4+ hours/week: 10pts
  - 2-4 hours/week: 8pts
  - <2 hours/week: 5pts

**Selection Process**:
1. Score all applications
2. Select top 15-20 by score
3. Ensure diversity across tech stacks
4. Send acceptance emails within 3 days

## Contingency Plans

### Low Participation
- Extend recruitment
- Offer additional incentives
- Reduce scope of testing

### High Bug Rate
- Pause recruitment
- Focus on stabilization
- Consider extending beta period

### Tester Dropoff
- Check in with inactive testers
- Adjust time commitments
- Recruit replacements if needed
