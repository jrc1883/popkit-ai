# PopKit Roadmap

**Last Updated:** 2026-02-12  
**Current Focus:** Stabilize v1.0 beta quality and simplify public repo maintenance.

## Principles

- Keep public documentation user-facing and low-noise.
- Keep implementation planning in issues/PRs (or private planning systems), not long-lived markdown snapshots.
- Prefer stable docs over status tables that go stale quickly.

## Near-Term Priorities

1. CI reliability and faster feedback loops.
2. Public docs quality in `packages/docs`.
3. Core hook safety and cross-platform consistency.
4. Packaging and release hardening for v1.0.
